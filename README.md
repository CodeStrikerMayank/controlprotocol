# CONTROL TOOL

A **Python-based terminal control system** designed to provide structured visibility, controlled file operations, and system-level insights through an interactive command-line interface.

This project focuses on **precision, modular design, and operational clarity**, prioritizing direct control and transparency over graphical abstraction.

---

## Overview

CONTROL TOOL is an interactive CLI application built in Python that enables users to inspect, manage, and organize system data efficiently from the terminal. It combines file system analysis, automation utilities, and system information into a single, cohesive control layer.

Rather than acting as a collection of isolated scripts, the system is designed as a **unified control environment**, where actions are deliberate, observable, and predictable.

---

## Key Capabilities

### 📁 Directory X-Ray (Tree View)

* Generates an **exact hierarchical tree structure** of any directory by providing its path.
* Displays files, subdirectories, and nesting clearly for deep inspection.
* Useful for understanding complex folder layouts quickly.

### 🔄 File & Folder Transfer

* Transfers files or directories **from one drive or location to another**.
* Preserves structure during transfer operations.
* Designed for controlled movement of data rather than blind copying.

### 🗂 Data Organization

* Automatically sorts files based on predefined rules.
* Helps maintain clean and structured directories.

### 🧹 Old File Cleanup

* Identifies and removes files based on **age thresholds**.
* Enables controlled cleanup without manual scanning.

### 🧠 Duplicate Detection

* Detects duplicate files to reduce redundancy.
* Assists in optimizing storage usage.

### 💽 Disk & System Insights

* Displays disk usage information.
* Retrieves essential system-level data for quick reference.

### ⚙️ Threshold-Based Control

* Allows actions to be triggered based on configurable thresholds.
* Adds logic-driven decision making to file operations.

### 🌐 App & Website Triggers

* Launches applications or websites directly from the terminal.
* Improves workflow efficiency without leaving the CLI.

---

## Design Principles

* **Terminal-First**: Built entirely for CLI interaction.
* **Modular Architecture**: Each feature is isolated and extendable.
* **Transparency**: Operations clearly communicate what is happening.
* **Control Over Automation**: Emphasis on intentional actions, not hidden behavior.
